Why Legacy Vault Protocol

8 pillars no competitor has

🔐

Zero Server Knowledge

Every document is AES-256-GCM encrypted before upload. The server stores only ciphertext, IVs, and tags — never plaintext.

📦

Private IPFS Storage

Documents live on a private swarm-keyed IPFS node. Content-addressed CIDs make tampering mathematically detectable.

⛓️

Blockchain-Anchored Audit

Every vault registration, manifest update, and status change is written to a private chain. The audit log cannot be altered.

⚖️

Multi-Proof Release

5 conditions must be cryptographically satisfied simultaneously: identity, death proof, attorney attestation, quorum, and waiting period.

👥

Guardian Quorum

N-of-M guardian approval enforced in code. No single party — executor, attorney, or beneficiary — can unilaterally release a vault.

🆔

W3C DID + VC 2.0

NIST SP 800-63-4 IAL 2/3 identity assurance for release-critical roles. Verifiable Credentials, not just email addresses.

💼

14-Model Data Architecture

Purpose-built schema: 6 roles, 10 asset categories, 13 document types, 22 audit actions, 9 claim states. Not a generic file tree.

🚀

Built for Web3

Crypto wallet registry, multi-chain asset support, DID-based governance — architected for the digital asset era, not scanned PDFs.
